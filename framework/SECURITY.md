# AI플랫마켓 보안 원칙 (각 독립 프로젝트 개발창 필독)

> 이 문서는 LifeMap, ItemZone, AI성장+마켓몰, 더착한쇼핑(몰) 등 모든 독립 프로젝트
> 개발창에 표준 헤더/Footer 파일과 함께 전달하는 보안 체크리스트입니다.
> 메인 AI플랫마켓 서버에서 실제 취약점 2건(피드백 API 개인정보 노출, 승인대기 업체
> 전체 노출)을 발견해 수정한 뒤 정리한 원칙입니다.

---

## 원칙 1. DB 문서를 통째로 API 응답에 펼치지 않기 (화이트리스트 방식)

```js
// ❌ 하지 말 것 — 문서에 나중에 민감한 필드가 추가되면 자동으로 노출됨
res.json({ id: doc.id, ...doc.data() });

// ✅ 이렇게 — 내보낼 필드를 항상 명시적으로 나열
const d = doc.data();
res.json({ id: doc.id, title: d.title, summary: d.summary, createdAt: d.createdAt });
```

## 원칙 2. "전체 조회"·"관리자용" 기능은 반드시 인증 확인

- 특정 사용자 한 명(`userId` 지정)의 본인 데이터 조회는 비로그인도 허용 가능
- 여러 명/전체 데이터 조회(`?all=true`, 관리자 대시보드용 API)는 항상 인증 미들웨어를 거칠 것
- 쿼리 파라미터 하나(`?all=true` 같은)로 필터를 우회할 수 없는지 반드시 점검

## 원칙 3. 새 API/컬렉션을 만들 때마다 이 질문을 스스로 하기

> "로그인 안 한 사람이 URL만 알면(또는 파라미터만 추측하면) 이 데이터를 볼 수 있는가?"

답이 "그렇다"면, 그 데이터가 실제로 공개되어도 괜찮은 것인지 다시 판단하고,
아니라면 인증을 추가합니다. 구현 후에는 인증 없이 직접 호출해보는 테스트를 한 번 해보세요.

## 이 프로젝트에서 특히 주의할 데이터

각 프로젝트 개발창에서 아래 항목이 있다면 위 3원칙을 적용해 직접 판단하세요
(구체적 필드 목록은 메인 창이 아니라 이 프로젝트 개발창에서 정하는 것이 정확합니다):

- **LifeMap**: 생년월일, 사주/성향/진로 분석 결과, 상담 내용 등 개인 식별·민감정보
- **ItemZone**: 사업 아이디어, 재무 정보, 아직 공개되지 않은 사업계획
- **공통**: 연락처(전화번호/이메일), AI Point 잔액·거래내역(price.md 참고), 결제 정보

## 참고 — AI Point / 결제 관련 원칙 (price.md 별도 문서)

가격·AI Point 정책은 이 문서가 아니라 저장소 루트의 `/price.md`를 따릅니다. 핵심만 요약:

- AI플랫마켓은 가격을 직접 결정하지 않음 — 모든 가격은 ChatDoumi AI Core 중앙 정책이 원본
- AI Point 잔액·차감·검증은 항상 서버 API(`/api/pricing/*`)를 통해서만 처리, 클라이언트에서 직접 계산 금지
- 결제/AI Point 승인 이전에는 AI 실행을 요청하지 않음
