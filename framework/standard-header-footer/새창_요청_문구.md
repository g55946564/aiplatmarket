# 다른 개발창(각 독립 프로젝트)에 전달할 요청 문구

아래 내용을 복사해서, LifeMap/ItemZone/AI성장+마켓몰/더착한쇼핑(몰) 등
각 독립 프로젝트를 개발하는 대화창 시작 시 함께 전달하세요.

---

## 복사해서 전달할 문구

> 첨부한 `AIPLATMARKET_HEADER_FOOTER.html` 파일은 AI플랫마켓 메인 사이트의
> 실제 GNB(헤더)와 Footer 코드를 그대로 추출한 참조 파일입니다.
>
> 이 프로젝트의 `index.html`을 만들 때 다음을 그대로 적용해주세요:
> 1. 파일 안의 `<style>` 블록 전체를 `<head>`에 붙여넣기
> 2. `<!-- HEADER -->` ~ `<!-- /HEADER -->` 사이 내용을 `<body>` 최상단에 붙여넣기
> 3. `<!-- FOOTER -->` ~ `<!-- /FOOTER -->` 사이 내용을 `<body>` 최하단에 붙여넣기
>    (본문 콘텐츠는 이 HEADER와 FOOTER 사이에 위치)
> 4. 맨 아래 `<script>` 블록을 `</body>` 직전에 붙여넣기
>
> GNB/Footer의 HTML 구조, class 이름, JS 함수명은 임의로 바꾸지 마세요 —
> 실제 메인 사이트와 시각적으로 100% 동일하게 유지되어야 합니다.
> 본문(Main Content) 영역만 이 프로젝트 특성에 맞게 자유롭게 구성하면 됩니다.

---

## 왜 이 방식인가

이전에는 `framework/assets/js/core.js`가 GNB/Footer를 "자동으로 재구성해서" 주입하는
방식을 썼는데, 실제 메인 사이트 코드를 그대로 복제한 게 아니라 별도로 요약 재구성한
것이어서 실제 GNB와 조금씩 어긋나고 드롭다운이 활성화되지 않는 문제가 있었습니다.

지금 방식(코드를 그대로 텍스트로 추출해서 각 프로젝트에 붙여넣기)은 "다른 버전이
생길 여지"가 없어 항상 실제 메인 사이트와 동일합니다. 대신, **메인 사이트의 GNB/Footer가
바뀔 때마다 이 참조 파일도 다시 추출해서 각 프로젝트 개발창에 재전달**해야 합니다.
(메인 개발창에서 "헤더/Footer 참조 파일 갱신해줘" 라고 요청하면 다시 만들어 드립니다.)
