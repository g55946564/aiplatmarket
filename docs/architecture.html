① GPT 질문 → 클로드 검토 의견
중간 그룹 폴더(platform/, commerce/…)를 지금 만들어야 하나요?
✅ 지금 만드세요. 라우트와 링크가 프로덕션에 깔리면 나중에 리팩토링 비용이 큽니다. 폴더는 만들되 파일은 필요할 때 채우면 됩니다. "10개 이하면 flat" 규칙은 SPA 내부 컴포넌트에 적용하고, URL 경로는 처음부터 계층형이 맞습니다.
Framework 역할을 어디까지 둘 것인가?
✅ GNB·Footer·Theme·Router·Auth·AI Core·공통 UI 모두 framework/ 에 넣되, 각 모듈이 직접 읽는 것이 아닌 index.js가 정적 파일로 서빙하는 구조로 설계하세요. 모듈은 CDN처럼 소비합니다.
독립 모듈의 기준 (AIR&D, 시뮬레이션, 아이템존 등)
✅ 각 모듈은 module.json (id, version, status, routes) + index.html 로 자기완결 구조. Framework의 공통 JS/CSS를 /framework/assets/ 경로로 include. 모듈 내부에는 framework 코드 없음.
URL 전략: path 방식 vs 서브도메인, 코드 수정 없이 둘 다 지원?
✅ index.js에서 req.hostname으로 서브도메인 감지 → 같은 파일 서빙. 내부 링크는 전부 절대 경로 (/platform/aird/). BASE_URL 환경변수로 서브도메인 전환 시 prefix만 바꾸면 됩니다.

③ URL 라우팅 전략
path/aird→platform/aird/index.html
path/platform/aird/→같은 파일
subaird.aiplatmarket.com→req.hostname 분기
코드 변경 없이 Nginx/Render 설정만으로 서브도메인 전환
④ 모듈 인터페이스 표준
module.json { "id": "aird", "version": "0.1.0", "status": "dev", "title": "AIR&D", "routes": ["/aird", "/platform/aird/"], "ai_features": ["analysis", "recommend"], "framework": "1.0.0" }
⑤ 개발 원칙 (클로드 제안)
1 Framework First — 모듈이 아닌 엔진 먼저
2 폴더 구조는 URL과 1:1 매핑
3 module.json 이 각 모듈의 계약서
4 미완성 모듈 → "업그레이드 중" 페이지
5 index.js 는 라우터 역할만, 로직은 framework/
