# 07. AI Simulation — 사업성·경쟁력·참여도·기여도·수익분배·투자 시뮬레이션

> 01번 문서(철학) §1.5 "공정한 기여도·수익분배"를 실제로 **어떻게 계산하는가**를 다루는 문서입니다.
> 이 시뮬레이션들은 AI아이템존(ItemZone)의 분석 단계, 그리고 향후 플랫폼 전체의 기여도 시스템에 적용됩니다.

## 7.1 시뮬레이션 5종 개요

| 종류 | 무엇을 예측/측정하나 | 현재 구현 위치 |
|---|---|---|
| 사업성 분석 | 시장 수요, 성장 가능성 | ItemZone 2단계 |
| 경쟁력 분석 | 유사 서비스 비교, 차별화 포인트 | ItemZone 4단계 |
| 참여도 분석 | 사용자가 플랫폼에서 얼마나 활동했는가 | 설계 단계 (7.3) |
| 기여도 분석 | 그 활동이 플랫폼 가치에 얼마나 기여했는가 | 설계 단계 (7.4) |
| 수익분배 분석 | 기여도를 실제 보상으로 환산 | 설계 단계 (7.5) |
| 투자 시뮬레이션 | 어느 투자 단계가 적합한지, 무엇을 준비해야 하는지 | ItemZone 8단계 |

## 7.2 사업성·경쟁력 분석 (이미 구현됨 — ItemZone)

AICore.process()를 통해 GPT/Claude/Gemini 중 하나가 다음 형식으로 응답하도록 프롬프트가 설계되어 있습니다:

```
시장 수요: (1줄)
경쟁 강도: (1줄)
성장 가능성: (1줄)
```

이 출력은 **정형화된 점수가 아니라 서술형**입니다. 향후 고도화 방향은 7.6을 참고하세요.

## 7.3 참여도 분석 (설계)

참여도는 "양"을 측정합니다. 예시 측정 항목:

| 활동 | 가중치(예시) |
|---|---|
| 커뮤니티 글 작성 | 3점 |
| 댓글/답글 | 1점 |
| 모임 만들기 | 5점 |
| 모임 참석 확정 | 2점 |
| 공동구매 모집 성사 | 8점 |
| ItemZone 아이디어 등록 | 5점 |
| 정보공유 글의 좋아요 10개 이상 획득 | +3점 (품질 보너스) |

> **가중치는 고정값이 아니라 운영 데이터로 주기적으로 재조정**합니다. 초기에는 위 표를 시작점으로 사용합니다.

## 7.4 기여도 분석 (설계) — 참여도와의 차이

참여도는 "얼마나 많이 했는가", 기여도는 **"그 활동이 실제로 플랫폼/다른 사용자에게
가치를 만들었는가"**를 봅니다. 예:

- 글을 10개 썼지만(참여도 높음) 아무도 안 읽었다 → 기여도는 낮음
- 글을 1개 썼지만 공동구매가 성사되어 12명이 참여했다 → 기여도는 높음

기여도 산출 공식(초안):
```
기여도 = Σ (활동별 참여도 가중치 × 실제 영향 지표)
실제 영향 지표 예: 댓글 수, 조회수/100, 공동구매 성사 인원, 모임 실제 참석 인원
```

이 산출은 **AI(통계적 1차 집계)가 하고, 이상치(어뷰징 의심)는 사람이 검토**합니다
(01번 문서 §1.5 "AI는 측정자, 사람은 검증자" 원칙).

## 7.5 수익분배 분석 (설계)

```
이번 달 분배 가능 재원 (광고수익+수수료 등의 일정 비율)
        ÷
전체 참여자의 기여도 총합
        ×
개인 기여도
        =
개인 분배액 (포인트 또는 실제 적립금)
```

**공개 원칙**: 매월 "분배 가능 재원"과 "기여도 총합"은 投명하게 공개하는 것을 목표로 합니다
(정확한 공개 방식·주기는 운영하며 확정).

## 7.6 투자 시뮬레이션 (이미 구현됨 — ItemZone 8단계, 고도화 방향)

현재는 "시드가 적합해 보입니다" 식의 서술형 안내입니다. 고도화 방향:
- 입력값(시장규모 추정, 초기 투자금, 예상 매출)을 받아 **간단한 손익분기점 계산**을 함께 제공
- 정부지원사업(5단계)과 투자 연결(8단계)의 데이터를 연계해 "지원사업으로 우선 검증 후 투자
  유치"같은 순서 추천

## 7.7 LifeMap과의 연계

LifeMap(AI인생·진로)의 "직업 적성/시장성 분석"은 이 문서의 사업성 분석과 같은 엔진(AI Core)을
공유하지만 프롬프트 컨텍스트가 다릅니다 (개인 진로 vs 사업 아이디어). 자세한 연동은
`docs/08_LIFEMAP_INTEGRATION.md`를 참고하세요.

## 7.8 구현 시 주의사항

- 모든 시뮬레이션 산출값은 **"확정된 사실"이 아니라 "참고용 추정"**임을 UI에 항상 명시합니다.
- 기여도/수익분배 수치를 실제 적립금으로 전환하는 기능은 **충분한 어뷰징 방지 로직 없이는
  배포하지 않습니다** (01번 문서 안티패턴 §1.7 참조).
