# 04. AI 개발 표준 — ChatGPT · Claude · Gemini · 향후 추가 AI

> 어떤 AI가 이 프로젝트에 합류하든, 동일한 규칙으로 이어서 개발할 수 있어야 합니다.
> 이 문서는 "AI마다 다르게 일하는 것"을 막기 위한 공통 행동 규약입니다.

---

## 4.1 모든 AI가 동일하게 지킬 5가지 규칙

1. **시작 시 `framework/memory/AI_BOOTSTRAP.md`부터 읽는다.** 추측으로 구조를 재구성하지 않습니다.
2. **AI Core를 우회하지 않는다.** 자신이 직접 GPT/Claude/Gemini API를 부르는 코드를 쓰지 않습니다 — 이미 그 자신이 AI Core 안의 한 Adapter일 수도 있다는 점을 기억합니다.
3. **framework/, ai-core/, docs/ 는 "제안"만 하고 직접 수정하지 않는다.** 구조 변경이 필요하다고
   판단되면 코드를 먼저 바꾸지 말고 사람(메인 개발자)에게 변경을 제안합니다.
4. **module.json 없이 모듈을 완성 처리하지 않는다.** index.html만 만들고 module.json을 빠뜻면
   `/api/ai-context`에 그 모듈이 누락되어 다른 AI/사람이 존재를 모르게 됩니다.
5. **불확실하면 "업그레이드 중"으로 처리한다.** 확신 없는 기능을 그럴듯하게 보이도록 더미 데이터로
   채우지 않습니다 (01번 문서 사람 중심 철학과 직결).

## 4.2 AI별 역할 분담 (현재 운영 기준, 고정된 규칙은 아님)

| AI | 강점 활용 영역 | 비고 |
|---|---|---|
| **Claude** | 대규모 코드 생성/수정, 버그 진단, 실제 브라우저 테스트(Playwright) | 메인 index.html/index.js 관리 |
| **ChatGPT (GPT)** | 시스템 아키텍처 검토, 전략 문서, 구조 제안 | 이 docs 체계의 초안 제안자 |
| **Gemini** | (현재) ChatDoumi AI Core의 Adapter 중 하나로만 참여 | 별도 전략 역할은 아직 없음 |
| **향후 AI** | 새 Adapter로 추가되거나, 특정 모듈(예: 시뮬레이션 엔진)의 전담 개발자로 합류 가능 | 합류 시 이 문서에 줄 추가 |

역할 분담은 "누가 더 잘하냐"의 자연스러운 결과이며, 강제 규칙이 아닙니다. 다만 **4.1의 5가지
규칙은 역할과 무관하게 모든 AI에게 동일하게 적용됩니다.**

## 4.3 새 대화창에서 작업을 시작하는 표준 절차

```
1. framework/memory/AI_BOOTSTRAP.md 읽기
2. framework/memory/MASTER_CONTEXT.md 읽기
3. (모듈 개발이면) framework/memory/MODULE_DEV_GUIDE.md 읽기
4. (필요하면) GET /api/ai-context 로 현재 실시간 상태 확인
5. 작업 시작
```

이 절차를 따르면 docs/ 9개 문서를 전부 읽지 않아도 80% 상황을 파악할 수 있습니다.
docs/는 "더 깊이 알아야 할 때"만 펼쳐보는 사전 역할입니다.

## 4.4 AI가 구조 변경을 "제안"하는 방법

구조(framework, ai-core, docs)를 바꿔야 한다고 판단되면:
1. 바로 코드를 고치지 않습니다.
2. 어떤 문제 때문에 어떤 변경이 필요한지 사람에게 설명합니다.
3. 승인 후, **docs/ 해당 문서를 먼저 갱신**하고, 그다음 코드를 변경합니다 (문서가 먼저, 코드가 나중).
   — 이 순서를 지켜야 "문서와 코드가 항상 같은 그림을 그린다"는 헌법의 신뢰가 유지됩니다.

## 4.5 AI 간 작업 인계 시 전달 형식 (권장)

다른 AI/대화창에 작업을 넘길 때는 다음을 함께 전달하는 것을 권장합니다:

> "{모듈명} 모듈을 framework/memory + docs 기준으로 개발했습니다.
> module.json status는 '{status}' 입니다. AI Core는 {엔진목록} 순서로 연결했습니다.
> 확인 부탁드립니다: public/{group}/{id}/"

이렇게 하면 받는 쪽(사람이든 AI든)이 별도 설명 없이 바로 검토에 들어갈 수 있습니다.
